Create a markdown live preview

by Sascha Brink

Buy this book

Problem

You’re using markdown somewhere in your application. Users, who edit the markdown templates, should see a live preview of the resulting html.

Solution

In this example, we implement the directive with the Showdown library, which is a popular markdown interpreter.

We create a directive ‘markdownPreview’ that has an isolated scope and takes a model as input attribute. As we create a new tag <markdown-preview>, we have to restrict it to element (E).

In the link function, we’re watching for a change of the model attribute and convert the markdown input to an html output. The result is pasted into the elements inner html.

angular.module('cookbookApp', [])
  .directive('markdownPreview', function() {
    var converter = new Showdown.converter();
    return {
      restrict: 'E',
      scope : {
        model: '='
      },
      link: function(scope, element) {
        scope.$watch('model', function(markdownText){
          var resultHtml = converter.makeHtml(markdownText || '');
          element.html(resultHtml);
        });
      }
    };
  });


<html ng-app="cookbookApp">
<head>
  <script src="https://ajax.googleapis.com/ajax/libs/angularjs/1.3.0/angular.min.js"></script>
  <script src="https://cdnjs.cloudflare.com/ajax/libs/showdown/0.3.1/showdown.min.js"></script>
  <script src="application.js"></script>
</head>
<body>
  <textarea ng-model="markdown" cols="60" rows="10"></textarea>
  <markdown-preview model="markdown"></markdown-preview>
</body>
</html>

Discussion

Quite often there are more solutions to one problem. If you want, you can also implement the markdown as a filter instead of directive. But be aware, that you have to include the ngSanitize module and use ng-bind-html. Otherwise, your resulting html will be escaped.
